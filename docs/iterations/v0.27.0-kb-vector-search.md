# v0.27.0 — 知识库向量检索 + Command Palette

## 概述

为 Moraya 知识库增加本地内容检索能力，在保持轻量级（Release 二进制 7.8MB）的前提下实现：

1. **BM25 关键词检索** — 自定义 BM25 实现，支持中英文分词，零外部依赖
2. **HNSW 向量检索** — `hnsw_rs` 纯 Rust HNSW 近似最近邻，Embedding API 对接 6 家 provider
3. **RRF 混合排序** — Reciprocal Rank Fusion 合并 BM25 + 向量结果
4. **Command Palette** — `Cmd+P` 文件搜索 / `Cmd+Shift+P` 命令面板 / 内容搜索
5. **Sidebar 内容搜索** — 输入即搜，关键词高亮，点击打开并滚动定位

**核心技术选型**：`hnsw_rs 0.3` + `anndists 0.1`（纯 Rust，~0.8MB 编译增量）+ 自定义 BM25 + 自定义 CJK 分词器

## 存储结构

### 向量索引（跟随知识库目录）

```
{kb_path}/.moraya/indexes/
  ├── hnsw.hnsw.graph   # HNSW 图结构（hnsw_rs file_dump 格式）
  ├── hnsw.hnsw.data    # HNSW 向量数据
  ├── vectors.bin       # 原始向量（f32 little-endian，用于 HNSW 重建）
  ├── chunks.jsonl      # chunk 元数据（每行一个 JSON 对象）
  ├── meta.json         # 索引元信息（版本、模型、维度、文件哈希）
  └── bm25.bin          # BM25 倒排索引（JSON 序列化）
```

**设计决策**：索引存储在知识库目录下的隐藏目录 `.moraya/indexes/` 中。优势：
- 知识库迁移/拷贝时索引跟随，无需重建
- 备份知识库即备份索引
- 删除知识库时索引自动清理
- `file.rs:429` 已通过 `starts_with('.')` 排除，无需修改 `read_dir_recursive`

## F1: Rust 后端 — 索引与检索引擎

### F1.1 核心模块

**文件**: `src-tauri/src/commands/kb.rs`（新增，~1000 行）

**数据结构**：
- `ChunkMeta` — chunk 序号、相对路径、所属标题、字节偏移、长度、预览文本
- `IndexMeta` — schema 版本、模型 ID、维度、文件哈希表、更新时间
- `SearchResult` — 文件路径、标题、关键词上下文片段、相关度、偏移量、来源类型
- `IndexStatus` — 索引状态、chunk 数、文件数、过期文件列表
- `IndexProgress` — 阶段（scanning/chunking/embedding/indexing/error/done）、进度
- `KBIndexState` — 索引缓存 + 并发控制锁
- `BM25Index` — 自定义 BM25 倒排索引（k1=1.2, b=0.75）

### F1.2 Tauri 命令

| 命令 | 用途 |
|------|------|
| `kb_index_files` | 全量/增量索引。`spawn_blocking` 执行分块，逐批 embedding，emit 进度事件 |
| `kb_index_single_file` | 单文件索引（保存时自动触发），全量重建 BM25 + HNSW |
| `kb_search` | 混合检索。mode: `"vector"` / `"bm25"` / `"hybrid"`（默认） |
| `kb_get_index_status` | 查询索引状态，扫描文件哈希报告过期文件 |
| `kb_delete_index` | 删除索引文件 + 清除缓存 |

### F1.3 文件收集

`collect_indexable_files(kb_path)` — 递归收集文本文件（最大深度 10）：
- **包含**：`.md`, `.markdown`, `.txt`, `.json`, `.yaml`, `.yml`, `.csv`, `.xml`, `.html`, `.rst`, `.org`, `.toml`, `.ini`, `.tex`, `.adoc` 等 18 种扩展名
- **排除**：`.moraya/`、隐藏文件、`node_modules`、`target`、`.git`、`__pycache__`、`.venv`、`dist`、`build`

### F1.4 文档分块

`chunk_document(content, file_path)` — 两级分块策略：

**优先：按 Heading 分块**
- 检测 `# ` / `## ` / `### ` 等 Markdown 标题
- 每个标题下的内容作为一个 chunk，标题存入 `ChunkMeta.heading`
- 超长 section 自动拆分为滑动窗口子 chunk

**兜底：固定大小滑动窗口**
- 无 Heading 时按 2000 字符（约 512 token）分块，300 字符重叠
- 在段落/句子/换行处优先断句
- 大文件（>1MB）使用 1000 字符窗口

**前处理**：剥离 YAML frontmatter，保留代码块

**CJK 安全**：所有字符串切片通过 `floor_char_boundary` / `ceil_char_boundary` / `safe_truncate` 辅助函数对齐到 UTF-8 字符边界，避免多字节字符截断 panic

### F1.5 Embedding API

`embed_texts_via_api` — 批量调用 Embedding API（每批 20 条，30s 超时）

**Endpoint 路由**（`embedding_endpoint` 函数自动检测版本路径）：

| Provider | Endpoint | 说明 |
|----------|----------|------|
| openai / deepseek / custom | `{base}/v1/embeddings` | OpenAI 兼容格式 |
| glm / doubao | `{base}/embeddings` | 已含 `/v3` 或 `/v4`，不再追加 `/v1` |
| gemini | `{base}/v1beta/models/{model}:batchEmbedContents` | Google 专有格式 |
| ollama | `{base}/api/embed` | Ollama 本地格式 |

**密钥解析**：直接访问 `AIProxyState.key_cache`（`pub(crate)`），复用 `ai_proxy.rs` 的 `ensure_secrets_loaded` 机制

**容错**：Embedding 失败自动降级为 BM25-only 索引，emit `error` phase 事件到前端

### F1.6 BM25 关键词检索

自定义实现（~200 行 Rust），无外部 crate：
- 参数：k1=1.2, b=0.75
- CJK 分词：每个汉字/假名/韩文音节作为独立 token
- 英文分词：空格+标点分割，过滤 <2 字符的 token
- 序列化：JSON 格式持久化

### F1.7 HNSW 向量检索

基于 `hnsw_rs 0.3` + `anndists 0.1`：
- 参数：`max_nb_conn=16`, `ef_construction=200`, `max_layer=16`, `DistCosine`
- 搜索参数：`ef_search=50`, `top_k * 2`（过采样后 RRF 合并）
- 持久化：`hnsw.file_dump()` 生成 `.hnsw.graph` + `.hnsw.data` 两个文件
- 增量更新：hnsw_rs 不支持单点删除，单文件索引时全量重建 HNSW

### F1.8 混合排序（RRF）

`rrf_merge` — Reciprocal Rank Fusion：
```
RRF_score(d) = Σ_{source} 1 / (60 + rank_source(d))
```
合并 BM25 和向量检索结果，按 RRF 分数降序排列

### F1.9 关键词上下文片段

`keyword_snippet(kb, chunk, query)` — 搜索时动态生成关键词上下文：
1. 从源文件读取 chunk 原文（使用 `offset` + `length`）
2. 搜索关键词位置（优先级：完整查询 > 分词 > CJK 单字）
3. 返回 ~200 字符以关键词为中心的上下文窗口
4. 未匹配时回退到 chunk 起始 200 字符

### F1.10 索引进度事件

通过 `app.emit("kb-index-progress", IndexProgress)` 发送到前端：
- 分块阶段在 `tokio::task::spawn_blocking` 中执行，每处理一个文件发一次事件
- Embedding 阶段每完成一个 batch（20 条）发一次事件
- 错误阶段发 `phase: "error"`，`file_name` 字段携带错误信息
- 完成阶段发 `phase: "done"`

### F1.11 文件变更检测

SHA-256 内容哈希（`sha2` crate），存于 `meta.json.file_hashes`：
- 增量索引对比当前哈希与存储哈希，跳过未变化的文件
- 优于 mtime：`touch`/`git checkout` 不触发误判

### F1.12 Cargo.toml 新增

```toml

anndists = "0.1"
```

## 错误处理

| 场景 | 处理 |
|------|------|
| AI Chat provider 不支持 embedding | 设置面板显示黄色提示，引导选择 embedding provider |
| Embedding API 404/错误 | emit `error` phase 事件，StatusBar 显示红色错误，5 秒消失 |
| Embedding API 失败 | 自动降级为 BM25-only 索引 |
| CJK 多字节字符截断 | `floor_char_boundary` / `ceil_char_boundary` 对齐到 UTF-8 字符边界 |
| 并发索引 | Mutex 保护，返回 "Indexing already in progress" |
| 索引文件损坏 | 捕获反序列化错误，提示重建 |
| KB 目录只读 | 返回权限错误提示 |
| 未配置 embedding 时右键索引 | 自动打开知识库设置面板 |
